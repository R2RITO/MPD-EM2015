El Usuario se tratará como super contexto, estará presente invariablemente en todos los dominios difusos utilizados.

No podrá haber inserts que no contengan un dominio especificado por cada una de las
dimensiones contextuales de las que depende el dominio difuso.
Es decir, si para PESO se añade TASK y ROL, no podrá haber inserts que no tengan
una "instancia" de "TASK".

Para cada entrada en la lista de dom de dim contextuales de un trapezoide (en catalogo contx)
cada dominio deberá ser del mismo usuario.

Un usuario no puede utilizar dominios de dim context que no sean definidos por el.
